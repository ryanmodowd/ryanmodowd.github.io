Algorithms

Info

            What are algorithms? Algorithms dictate a certain series of moves to alter the rubik's cube in a specific way. Nearly all the methods available to solve the rubik's cube use algorithms in one way or another. This is because the rubik's cube is an impossibly hard puzzle to solve based off of pure intuition. There is almost an endless amount of rubik's cube algorithms out there to make endless adjustments to the cube, but there are certain ones that are helpful to solve the cube while using a method. This page is here to be a reference for many helpful algorithms.

            What should you know before learning any algorithms?

                        • Algorithm Notation. This is discussed below.

                        • Which method you want to learn. Many algorithms are specific to a certain method, or portion of the method.

Algorithm Notation

            What use would algorithms be, if they could not be effectively conveyed to those who needed to use them? They would be entirely useless, so to prevent that issue, a language was created allowing algorithms to be written in a simplistic form so that those learning how to solve the cube could easily read and learn the algorithms they need to.

            The rubiks cube has 6 faces, and in algorithm notation they are called:

                        • Up, Down, Left, Right, Front, Back.

            For simplicity, when notating, the first letter of each is used:

                        • U, D, L, R, F, B.

            When any of these letters appears in algorithm notation, it signals that the solver should rotate that side 90° clockwise. Sometimes though, an algorithm may require a rotation of a side 90° counterclockwise, in which case the corresponding letter is followed by an apostrophe. In yet other cases, a side may be required to rotate 180°, in which case the number 2 preceeds the letter designating which face to turn. So for example:

                        • L R' U2 D2 F L'

            This notation signifies that the solver should rotate the left side clockwise 90°, the right side counterclockwise 90°, the up side 180°, the down side 180°, the front side 90° clockwise, and the left side 90° counterclockwise. It should be noted that when using any of these letters in the notation, the center facing you should always be the same color, and should never rotate away from you.
